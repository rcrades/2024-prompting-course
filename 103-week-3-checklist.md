# Week 3: Graduation - Your Full-Stack Capstone Project

Congratulations on reaching the final week of Prompting 101! This week, you'll build a complete full-stack application that showcases everything you've learned. **You get to choose your own project** from a curated list, or propose your own idea!

## Final Project Overview

Your capstone project will demonstrate:
- Mastery of the v0 → GitHub → Cursor workflow
- Ability to add a backend (Python or Node.js)
- Working with APIs and data persistence (JSON or simple database)
- Professional-grade prompting and iteration skills
- A portfolio-worthy final product

## Choose Your Adventure: Project Options

### Option 1: Personal Project Hub 🎨
**Description**: A dashboard that showcases all your course projects in one place

**Features**:
- Landing page with links to all your projects
- Project cards with screenshots and descriptions
- About section highlighting your AI-assisted development journey
- Backend API to manage project data
- Contact form that saves to JSON file

**Great for**: Portfolio showcasing, linking in job applications

### Option 2: Recipe Manager 🍳
**Description**: Full-stack app to save, organize, and search recipes

**Features**:
- Add/edit/delete recipes with ingredients and instructions
- Category filtering (breakfast, lunch, dinner, dessert)
- Search functionality
- Save recipes to JSON file via Python/Node backend
- Image upload for recipe photos (optional)

**Great for**: Practical everyday use, demonstrates CRUD operations

### Option 3: Expense Tracker 💰
**Description**: Track personal expenses and visualize spending

**Features**:
- Add expenses with category, amount, date
- View expense history
- Filter by date range or category
- Simple charts/visualizations of spending
- Backend API to persist data in JSON

**Great for**: Useful life skill, teaches data visualization

### Option 4: Study Flashcard App 📚
**Description**: Create and study flashcard sets

**Features**:
- Create flashcard decks on different topics
- Flip card animation to reveal answers
- Track which cards you've mastered
- Backend to save decks and progress
- Spaced repetition algorithm (advanced)

**Great for**: Meta learning tool, demonstrates stateful interactions

### Option 5: Link Aggregator (Linktree Clone) 🔗
**Description**: Personal link-in-bio page with analytics

**Features**:
- Custom branded landing page
- Multiple links with titles and icons
- Click tracking for each link
- Admin panel to add/edit/remove links
- Backend API to manage links and track clicks

**Great for**: Personal branding, useful for social media

### Option 6: Propose Your Own! 💡
**Have a different idea?**
- Must include frontend (built with v0) + backend (Python or Node.js)
- Must persist data (JSON file minimum)
- Must demonstrate full workflow: v0 → GitHub → Cursor
- Share your proposal in Discord for feedback before starting!

## Project Development Checklist

### Phase 1: Planning & Design (Day 1)
- [ ] Choose your project from the options above
- [ ] Write out all the features you want to include
- [ ] Sketch a rough layout (pen and paper is fine!)
- [ ] Plan your data structure (what will be saved in JSON?)
- [ ] Share your plan in Discord for feedback

### Phase 2: Frontend Development with v0 (Days 1-2)
- [ ] Craft a detailed prompt for v0 describing your app
- [ ] Build the initial UI in v0
- [ ] Iterate on the design until it looks great
- [ ] Test the UI interactions (even without backend)
- [ ] Export the v0 code
- [ ] Push to a new GitHub repository

### Phase 3: Backend Setup (Days 2-3)
- [ ] Choose your backend: Python (Flask) or Node.js (Express)
- [ ] Clone your repo in Cursor AI
- [ ] Use Cursor to help set up a simple backend:
  - [ ] Create API endpoints (GET, POST, PUT, DELETE)
  - [ ] Implement JSON file reading/writing
  - [ ] Add error handling
  - [ ] Test API endpoints with Postman or curl
- [ ] Connect frontend to backend API

### Phase 4: Integration & Features (Days 3-4)
- [ ] Connect all frontend actions to backend API calls
- [ ] Implement data persistence (save to JSON file)
- [ ] Test full CRUD operations:
  - [ ] Create new items
  - [ ] Read/display existing items
  - [ ] Update existing items
  - [ ] Delete items
- [ ] Add data validation and error handling
- [ ] Make sure data persists after page refresh

### Phase 5: Polish & Enhancement (Days 4-5)
- [ ] Improve UI/UX based on testing
- [ ] Add loading states and error messages
- [ ] Ensure responsive design works on mobile
- [ ] Add at least 2 "nice-to-have" features
- [ ] Fix any bugs with Cursor's help
- [ ] Optimize performance

### Phase 6: Documentation & Presentation (Days 5-6)
- [ ] Write a comprehensive README:
  - [ ] Project title and description
  - [ ] Screenshots/GIFs of the app in action
  - [ ] Features list
  - [ ] Technologies used (v0, React, Python/Node, etc.)
  - [ ] Setup instructions (how to run locally)
  - [ ] API documentation
  - [ ] Your prompting journey and learnings
- [ ] Create a presentation (5-7 slides) covering:
  - [ ] What you built and why
  - [ ] Demo of key features
  - [ ] Challenges you overcame
  - [ ] How you used AI tools (v0, Cursor, ChatGPT/Claude)
  - [ ] What you learned
  - [ ] Next steps/future improvements
- [ ] Optional: Deploy your app (Vercel for frontend, render.com for backend)

### Phase 7: Share & Celebrate! (Day 6)
- [ ] Share your project in Discord
- [ ] Give feedback on at least 2 other students' projects
- [ ] Present during the final Zoom call
- [ ] Add project to your portfolio/resume
- [ ] Celebrate your achievement! 🎉
